# ADR 0002：节点配色以 NODE_COLORS 为单一真源

- **日期**：2026-07-07
- **状态**：采纳
- **关联 feature**：feature-003-design-system

## 背景
feature-003 spec 的 AC1 给出一版试探性色板(page=#8b5cf6、component=#06b6d4、api=#10b981、handler=#f59e0b、service=#ef4444、db=#ec4899)。但项目 Cytoscape 画布实际渲染取自 `packages/shared` 的 `NODE_COLORS`(17 种,page=#6366f1、component=#3b82f6、service=#8b5cf6、db_model=#ec4899…),二者不一致。

AC2 是硬要求:图例(Legend)配色必须与画布**实际渲染**一致。若图例照 AC1 的试探色板渲染,反而会与画布不符,违背 AC2 与「杜绝不一致」的设计目标。

## 决策
以 `packages/shared/src/constants/nodeColors.ts` 的 `NODE_COLORS`(经 `src/lib/nodeConfig.ts` re-export)为**唯一真源**:
- Cytoscape stylesheet、图例 Legend 均从 `NODE_COLORS` / `NODE_EMOJI` 取值,不各自硬编码。
- `tailwind.config.js` 的 `theme.extend.colors.node.*` 仅**镜像**这些同值,供以 class 引用配色时同样一致(当前无组件以 node-* class 取色,属预留)。
- AC1 中试探色板与真源冲突处,一律以真源为准;并覆盖全部 17 种节点类型,而非只 6 种。

## 影响
- 图例与画布 100% 同源,视觉统一。
- 未改变任何节点类型语义或数量(仍 17 种),符合 Non-Goals。
- 后续若要调色,只改 `NODE_COLORS` 一处,画布 + 图例 + tailwind token 需同步(tailwind 为手工镜像,改色时一并更新)。
