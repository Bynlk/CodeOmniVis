# ADR-0004 表现层从 0 重写，保留数据管线与契约

日期：2026-07-07
状态：已接受

## 背景

feature-004（布局重构）与 feature-010（布局层级重设计）采取的是对旧组件的增量补丁（保留并修改现有可视组件）。虽然两者都通过了验证，但用户明确判定这种做法是「偷懒」，要求「从 0 开始重新写前端页面」。前端的痛点（App.tsx 状态中心、Tab 过载、浮层争抢、图例缺失、搜索双轨、无响应式、无 a11y）是结构性的，补丁无法根治。

## 决策

将整个**表现层**推倒重写（feature-011），不复用旧可视组件代码；同时**保留健康的数据管线与契约**。

### 推倒重写（REBUILD）
- App 外壳（去除状态中心，只留布局编排）。
- 布局系统（全新 AppShell，CSS Grid 语义分区，z 分层，响应式）。
- 全部可视组件（Header/Sidebar/Legend/NodeDetailPanel/CommandPalette/Tab 面板组/SettingsDrawer/Tooltip/WsStatusIndicator/FreshnessBadge）。
- 设计 token / 样式（index.css 去 reset，tailwind token 扩展）。

### 保留（PRESERVE）
- packages/shared 数据契约与 12 个 server API（LOCKED，100% 兼容）。
- services/（API 解耦层）。
- hooks/（React Query + WebSocket 服务端状态层）。
- utils/graphTransform.ts、lib/{nodeConfig,edgeConfig,i18n,cytoscapeContext,selectionContext}。
- store/uiStore.ts（UI 状态层，重接线不重写）。
- 技术栈（constitution，不引入新依赖）。

## 后果
- 好处：信息架构与交互得到结构性重塑，真正满足用户「从 0 重写」的意图，同时不引入契约回归风险。
- 代价：大量组件需重写与重新测试；旧组件及其专属测试需下线/重写。
- feature-004 / feature-010 的布局成果被本次重写取代，其设计思路（栅格分区、z 分层、浮层仲裁）被吸收进新 AppShell。

## 与已有 ADR 的关系
- ADR-0001（技术栈）：继续遵守，不引入新框架/状态库。
- ADR-0002（节点配色单一真源）：继续遵守，Legend 与 Cytoscape 均从 nodeConfig 取色。
- ADR-0003（布局层级）：其层级仲裁规则被新 AppShell 吸收。
