# ADR 0003：布局层级重设计（feature-010）与 feature-004 的关系

- **状态**：已采纳（Accepted）
- **日期**：2026-07-07
- **相关**：feature-004-layout-refactor（已归档）、feature-007-responsive、feature-010-layout-hierarchy

## 背景
feature-004 的 spec 声称建立「顶栏/侧栏/画布/详情四区栅格」，AC3 要求「无重叠错位」。实现中 TabPanel 已 dock 化（md:static 进栅格），但 **NodeDetailPanel 始终保持 absolute 浮层**（`absolute right-0 top-0 bottom-0 z-20`），从未真正进入栅格。用户实测报告「不同页面重叠」，代码核查确认两处结构性缺陷：
1. 详情浮层与分析 dock 轨道范式冲突 → 二者可空间重叠；
2. 详情面板 absolute 锚点为视口（父容器无 relative）→ 压住 Header。
此外 z-index 全站散落魔法值、浮层间无互斥。

## 决策
不重开 feature-004（其 dock 化成果保留、验证有效），而是新增 **feature-010** 补完并叠加治理：
1. 详情面板真正进主区 CSS Grid 右轨，去除相对视口的 absolute 定位；
2. 详情面板与分析 dock 取**互斥**占用同一右轨（打开其一自动收起另一），而非并存双轨——更契合 brief G1/G2「3 秒定位、主次分明」；
3. z-index 收敛为 tailwind 语义 token，且 tooltip 层级严格低于 modal；
4. uiStore 新增单一浮层仲裁（Cmd+K / 设置 / 移动抽屉互斥，模态期间抑制 tooltip）。

## 理由
- feature-004 的 dock 与 feature-007 的响应式类是有效资产，回退代价大且无收益；缺陷集中在「详情面板未入栅格 + 无 z 治理 + 无浮层仲裁」，范围可隔离为增量 feature。
- 互斥而非并存：右侧同时两块面板正是用户观感差的根源之一；主次分明优先。

## 影响
- 改动文件：App.tsx、NodeDetailPanel.tsx、TabPanel.tsx、uiStore.ts、tailwind.config.js、NodeTooltip.tsx。
- 兼容性：不动数据契约 / 后端 / Cytoscape / services；不加依赖。
- 若未来需要「详情+分析并存」，须另开 ADR 推翻本决策的互斥条款。
