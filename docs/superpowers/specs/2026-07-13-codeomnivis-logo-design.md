# CodeOmniVis Logo 设计规范

## 1. 设计目标

Logo 应体现 CodeOmniVis 是一个专业、克制、可信的开发工具，而不是消费级 SaaS、游戏产品或加密货币项目。

标志需要同时满足：

- 远看是结构清晰的工具图标；
- 近看能识别 C、O、V 三个字母；
- 呼应代码拓扑、全局视野与链路追溯；
- 不使用渐变、阴影、发光或装饰性节点；
- 能稳定用于 Web 顶栏、侧边栏、favicon、CLI、README 和 GitHub 头像。

## 2. 核心概念

采用“A · 紧凑核心 / Clean V”方向：

- 外层开放圆弧构成 C，表达对代码库的全局扫描与可见边界；
- 内层圆环构成 O，表达 Omni / 全局视野；
- 内部切角路径构成 V，表达 Visualization 与可追溯的调用方向；
- V 不使用端点圆点、方点或其他流程图装饰。

标志由三条独立路径组成，保持单一职责，便于按尺寸降级和按主题换色。

## 3. 几何规范

基础坐标系为 `64 × 64`，透明背景。

### C 外环

- 使用圆弧而非多边形；
- 参考路径：`M48 15A22 22 0 1 0 48 49`；
- 标准线宽：`5.5`；
- 端点为圆头，形成友好但不柔弱的外轮廓；
- 右侧开口必须清晰，避免与 O 混成双圆图案。

### O 内环

- 圆心：`33, 32`；
- 半径：`11`；
- 标准线宽：`3.5`；
- 使用品牌强调色；
- 与 C、V 之间保留明确负空间。

### V 路径

- 参考路径：`M26 26.5L33 38L40 26.5`；
- 标准线宽：`3.2`；
- 使用直线与锐利转角；
- 不添加端点、节点、箭头或断线；
- V 保持在 O 内部，不穿出 O 或 C 的开口。

## 4. 配色规范

最终品牌强调色为 Cobalt Signal。

| 用途 | 深色界面 | 浅色界面 |
|---|---|---|
| C / V / 主字标 | `#F4F7FB` | `#141922` |
| O 内环 / 字标大写 O | `#6F83FF` | `#566BE8` |
| 次级说明文字 | `#718095` | `#657084` |

约束：

- 不使用渐变；
- 强调色只用于 O，不扩散到 C、V 或整段字标；
- 单色场景中三条路径统一使用同一颜色；
- 强调色占标志视觉面积不超过约 25%。

## 5. 字标规范

正式名称始终写作 `CodeOmniVis`。

- 使用中性现代无衬线字体，优先采用项目现有的 Inter 字体栈；
- 字重建议 `650`，浏览器不支持可变字重时降级为 `600`；
- 字距建议 `-0.02em` 至 `-0.03em`；
- 仅 `Omni` 的大写 O 使用品牌强调色；
- `Code` 中的小写 o 保持中性色；
- 其余字符不得分段着色；
- 图标与字标的标准水平间距为图标宽度的约 `0.18–0.22`。

产品界面中的字标应由组件和字体渲染，不把字体依赖写进基础标志 SVG。这样既保持清晰度，也避免 SVG 文本在不同系统中换字体。

## 6. 尺寸与变体

### 标准标志

- 适用尺寸：`24px` 及以上；
- 使用完整 C + O + V；
- Web 顶栏推荐 `24px`；
- 侧边导航和 GitHub 头像推荐 `32–64px`。

### 极小尺寸

- 适用尺寸：`16px`；
- 可省略 O 环，只保留加粗的 C + V；
- C 建议线宽约 `7`，V 建议线宽约 `6`（以 64 坐标系计）；
- 不尝试在 16px 中保留细 O，避免糊成一团。

### 单色标志

- 三条路径使用同一颜色；
- 用于终端、打印、特殊 GitHub 物料和不支持多色的环境；
- 不通过透明度制造第二层级。

### 横向锁定组合

- 图标在左，`CodeOmniVis` 字标在右；
- 仅大写 O 与图标 O 同色；
- 可选次级说明 `Architecture intelligence`，但产品顶栏不默认展示该说明。

## 7. 资产与组件边界

实现阶段应产出：

- 深色背景标准标志 SVG；
- 浅色背景标准标志 SVG；
- 单色 SVG；
- 16px 简化 favicon SVG；
- 可根据主题换色的 React `BrandLogo` 组件；
- 横向字标由 `BrandLogo` 组件组合图标和文本，不依赖 SVG `<text>`。

图形资产只负责标志路径；组件负责尺寸、主题、字标、可访问名称和布局间距。

## 8. 使用限制

- 不旋转标志；
- 不拉伸或改变 C/O/V 的相对比例；
- 不给 V 添加节点点位；
- 不给 O 增加发光或渐变；
- 不把整个 `Omni` 或 `COV` 字符串染成强调色；
- 不在小尺寸中强行保留无法清晰显示的细节；
- 图标四周安全区不得小于图标宽度的 `12.5%`。

## 9. 验收标准

- `24px`、`32px`、`64px` 下可辨识完整 C/O/V，`16px` 简化版可辨识 C/V 并与标准标志保持家族一致性；
- 深色和浅色背景对比度清晰；
- 单色版本不依赖颜色仍能识别 C/O/V；
- SVG 没有渐变、滤镜、阴影、位图或外部字体依赖；
- SVG 具有有效 `viewBox`，路径不会超出安全区；
- Web 顶栏和侧边栏不再显示旧的 `CO` 文本方块；
- favicon 使用 16px 简化结构；
- `CodeOmniVis` 仅大写 O 使用强调色；
- Logo 组件提供明确的可访问名称，并可在纯装饰场景隐藏于辅助技术。
