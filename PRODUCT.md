# Product

## Register

product

## Users

CodeOmniVis primarily serves developers maintaining full-stack repositories and developers who use AI coding agents through MCP. They work inside an implementation or debugging flow and need a trustworthy map of the repository without leaving the local development environment.

## Product Purpose

CodeOmniVis turns a repository into an explorable architecture model. The first job is to understand the system at a glance; the next jobs are locating a module or file, identifying risks, and tracing a page or component through APIs and services to data models. Success means a developer can answer those questions without reading dozens of files first.

## Brand Personality

Precise, restrained, trustworthy. The interface should feel like a focused IDE workbench: dense when the task requires it, quiet when it does not, and explicit about data freshness and uncertainty.

## Anti-references

- No gradients, decorative glass, oversized cards, or playful dashboard styling.
- No emoji-led navigation or chat-first AI product framing.
- No visual noise that competes with the graph.
- No opaque automation that hides whether a relationship is certain or inferred.

## Design Principles

1. Start with system shape, reveal implementation detail progressively.
2. Keep one stable workspace while views change around the same graph.
3. Make system status, confidence, and source location explicit.
4. Prefer familiar IDE and observability-tool patterns over novel controls.
5. Let AI consume architecture through MCP; keep the human Web UI focused on exploration.

## Accessibility & Inclusion

Target WCAG 2.1 AA. All primary workflows must support keyboard navigation, visible focus, non-color state cues, reduced motion, and readable contrast. Dark theme is the default.
